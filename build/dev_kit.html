Создание новой страницы

Создание новой страницы подразумевает создание нескольких файлов и написание некоторых зависимостей
1. в папке pug/pages создать новый файл и назвать его соответственно тех. заданию. (например page.pug). Скопировать шаблон вида (или из любой рядом лежащей страницы)
extends ../layout/layout.pug
block vars
    - var title = 'Digiseller';
block header
    include ../includes/header.pug
block content
    include ../includes/content-file_name.pug
block footer
    include ../includes/footer.pug
в папке includes создать контентную часть страницы, которая должна иметь название вида content-file_name.pug
2. Файл стилей для контентной части страницы создается в папке static/import/pages/page.scss
Если необходимо создание стилей компонента, тогда создается файл в static/import/components/component.scss
В файле main.scss произвести импорт необходимого файла соответственно категории (компонент, страница, шаблон)

Правила именования элементов

Все кастомные классы, написанные верстальщиком записаны через нижнее подчеркивание. Для удобства выделения этих классов программистами.В проекте не используется BEM или любая другая методология названия классов и структуры.
Названия идентификаторов - есть отображение их сущности (seller_name_block, purchase_title, etc). Кроме случаев абстракции компонентов, которые будут повторяться.В случае когда компонент не отображает содержимого (напрпимер кнопки различного содержания, но одинакового вида: сохранить, отменить, submit), допустимо использовать название цвета.

Табы

Для создания собственных табов необходимо ознакомиться с файлом mixins.pugи узнать как в целом пишутся миксины в pug + базовые знания JS
В функции +tab()первый аргумент - массив названий вкладок, второй - числовое значение активной вкладки (начиная с нуля)
Для создания вкладки с числовым значением, в массиве названий должен быть многомерный массив, состоящий из строки и числа+tab-list(['Tab1', ['Tab2', 192]], 0)

Иконки

Для вставки иконок используем миксин +icon('')где аргументом является название файла svg, например+icon('webmoney')с миксином для иконок также можно ознакомиться в файле mixins.pug
Все новые иконки добавляются в папку svg, после чего необходимо произвести сборку проекта, чтобы сформировался спрайт.

Таблицы

Таблицы в проекте - это одна из основ. В них хранится большинство данных, которые получает пользователь. Особенно важно сделать их одинаковыми настолько, насколько это возможно. Не только визуально, но и структурно, с точки зрения верстки.
Все таблицы получают размеры из шаблона, которые определяется тегамиcolgroupи col. Где количество колонок должно совпадать с количеством колонок таблицы.
colgroup
  col
  col
thead
  tr
    th ID
  tr
    th название
tbody
  tr
    td 1234567
  tr
    td Продукт 1
Учитывая огромное количество таблиц в проекте и их неоднозначную ширину от страницы к странице - использование шаблона колонок позволит избавитьсяот инлайновых стилей в каждой из колонок, когда потребуется задать фиксированную ширину одной из колонок. Ширина колонки задается стандартным образом
colgroup
  col(style="width: 100px")
  col(style="width: 34px")
По умолчанию все колонки таблицы делят ширину таблицы поровну. Если одна из колонок получает фиксированную ширину, то остальные колонки будут делить оставшееся место поровну.
