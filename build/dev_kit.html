Создание новой страницы

Создадим несколько файлов:
1. в папке pug/pages создать файл и назвать его соответственно тех. заданию. (из этого файла будет происходить компиляция). Скопировать шаблон вида (или из любой рядом лежащей страницы)
extends ../layout/layout.pug
block vars
    - var title = 'Digiseller';
block header
    include ../includes/header.pug
block content
    include ../includes/content-file_name.pug
block footer
    include ../includes/footer.pug
2. в папке includes создать контентную часть страницы, которая должна иметь название вида content-file_name.pug
3. Файл стилей для контентной части страницы создается в папке static/import/pages/page.scss
Если необходимо создание стилей компонента, тогда создается файл в static/import/components/component.scss
В файле main.scss произвести импорт необходимого файла соответственно категории (компонент, страница, шаблон)

Правила именования элементов

Все кастомные классы, написанные верстальщиком записаны через нижнее подчеркивание. Для удобства выделения этих классов программистами. В проекте не используется BEM или любая другая сторонняя методология названия классов и структуры.
Названия идентификаторов - есть отображение их сущности (seller_name_block, purchase_title, etc). Кроме случаев абстракции компонентов, которые будут повторяться. В случае когда компонент не отображает содержимого (напрпимер кнопки различного содержания, но одинакового вида: сохранить, отменить, submit), допустимо использовать название цвета.

Для создания табов необходимо ознакомиться с файлом mixins.pugи узнать как в целом пишутся миксины в pug + базовые знания JS
В функции +tab()первый аргумент - массив названий вкладок, второй - числовое значение активной вкладки (начиная с нуля)

Таблицы

Таблицы в проекте - это одна из основ. В них хранится большинство данных, которые получает пользователь. Особенно важно сделать их одинаковыми настолько, насколько это возможно. Не только визуально, но и структурно, с точки зрения верстки.
Все таблицы получают размеры из шаблона, которые определяется тегамиcolgroupи col. Где количество колонок должно совпадать с количеством колонок таблицы.
colgroup
  col
  col
thead
  tr
    th ID
    th Название
tbody
  tr
    td 1234567
    td Название товара
  tr
    td 1234567
    td Название товара
Учитывая огромное количество таблиц в проекте и их неоднозначную ширину от страницы к странице - использование шаблона колонок позволит избавиться от инлайновых стилей в каждой из колонок, когда потребуется задать фиксированную ширину одной из колонок. Ширина колонки задается стандартным образом
colgroup
  col(style="width: 100px")
  col(style="width: 34px")
По умолчанию все колонки таблицы делят ширину таблицы поровну. Если одна из колонок получает фиксированную ширину, то остальные колонки будут делить оставшееся место поровну.
